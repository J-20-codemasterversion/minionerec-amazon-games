# 推荐系统流水线 V3 — 完整流程说明

## 总体架构

```
数据加载 → 交互矩阵构建 → 训练/测试划分 → 多路召回 → 粗排(GBT) → 精排(FM-BPR) → 重排序 → 评估
```

---

## 1. 数据加载（用户中心策略）

### 1.1 Pass 1：轻量扫描（提取 uid/iid/rating/ts）

- **输入**：`All_Amazon_Review_5.json`（约 105GB）
- **扫描范围**：前 10M 行
- **方式**：逐行二进制读取，`orjson` 快速解析
- **目的**：构建 `user_tuples`（用户→交互列表）和 `item_user_lines`（商品→用户行号）的内存索引

### 1.2 选取活跃用户

- 从所有用户中筛选交互 ≥ 5 条的用户
- 取交互数最多的 **top 10,000** 用户作为"种子用户"
- 每用户按时间排序，截取前 **8 条**交互
- 收集种子用户涉及的所有商品 → **~40,586 个商品**

### 1.3 热门商品补充

- 被 ≥ 2 个种子用户评价过的商品标记为"热门商品"（~14,168 个）
- 为每个热门商品**最多补充 30 条**其他用户的评论（提供协同过滤共现信号）
- 补充评论约 **30-40 万条**

### 1.4 Pass 2：精准回读

- 根据 Pass 1 收集的行号集合，精准回读对应行
- 只解析命中行号的行，跳过其余行
- **输出**：完整的 `reviews` 列表（~40-50 万条评论）

### 1.5 商品元数据加载

- **输入**：`All_Amazon_Meta.json`
- 快速跳过前 1200 万行（已知无匹配）
- 用 bytes 级 ASIN 预过滤，只对命中行做完整 JSON 解析
- 加载商品的 title/brand/category/price/rank/also_buy/also_view 等字段
- **覆盖率**：约 85%

---

## 2. 交互矩阵构建 + 统计特征

- 构建 `uid2idx` / `iid2idx` 双向映射
- 构建 `user2items` / `item2users` 交互集合
- 计算统计特征：
  - 用户侧：`user_avg_rating`, `user_rating_cnt`, `user_rating_std`
  - 商品侧：`item_avg_rating`, `item_rating_cnt`
  - 全局：`global_avg`

---

## 3. 训练/测试集划分（Leave-Last-One-Out）

- 按用户分组，每个用户的**最后一条交互**作为测试
- 交互 ≥ 3 条的用户：前 N-1 条训练，最后 1 条测试
- 交互 < 3 条的用户：全部进训练集（不产生测试样本）
- 构建训练集交互索引：`train_user2items_idx`, `train_item2users_idx`

---

## 4. 多路召回（5 通道）

### 4.1 热门召回（HotRecall）
- 按商品被评价的用户数排序，返回全局热门商品

### 4.2 ItemCF 召回
- 基于 item-item 协同过滤
- 构建 item 相似度矩阵（top_k_sim=50）
- 根据用户历史交互物品，召回相似物品

### 4.3 UserCF 召回
- 基于 user-user 协同过滤
- 构建 user 相似度矩阵（top_k_sim=30, max_users=50,000）
- 找到相似用户，推荐其喜欢的物品

### 4.4 Swing 召回
- Swing 模型，考虑用户对之间的交互重叠
- alpha=0.5, top_k_sim=50, max_items=15,000

### 4.5 内容召回（ContentRecall）
- 基于品牌/品类/标题关键词倒排索引
- 匹配用户历史偏好的品牌、品类、关键词

### 4.6 多路融合

- 4 路融合权重：hot=0.1, itemcf=0.3, usercf=0.2, swing=0.4
- Content 通道权重：0.25
- **多路命中加分**：被多个通道召回的商品额外加 10% × 通道数
- 每用户召回 **top 1000** 候选
- 输出格式：`(item_idx, recall_score, sources_set)`

## 6. 粗排（GBT — Pointwise + Pairwise 负采样）

### 训练
- **模型**：`GradientBoostingClassifier`（sklearn）
- **训练样本**：最多 50,000 条正负样本对
- **正样本**：用户实际交互过的商品
- **负采样**：按热度 ^0.75 加权采样（Hard Negative）
- **特征**：25 维扩展特征（归一化）
- **超参数**：
  - n_estimators=200, max_depth=4, learning_rate=0.08
  - subsample=0.8, min_samples_leaf=20

### 推理
- 对每用户的 1000 个召回候选批量打分
- 取 **top 100** 进入精排

---

## 7. 精排（FM — BPR Pairwise 损失）

### 模型结构
FM (Factorization Machine) 预测分数：

```
score = w0 + w_user[u] + w_item[i] + w_fine · x
        + 0.5 * [||v_user[u] + v_item[i] + V_fine·x||² - (v_user[u]² + v_item[i]² + V_fine²·x²)]
```

- `K_FM = 8`（隐向量维度）
- 用户/商品各有独立的一阶偏置和隐向量

### BPR 训练
- **训练对**：最多 100,000 个 (user, pos_item, neg_item) 三元组
- **负采样**：同粗排，热度 ^0.75 加权
- **优化器**：Mini-batch SGD（batch_size=1024）
- **Epochs**：15
- **学习率**：0.005
- **正则化**：L2，w 正则 0.01，v 正则 0.01
- **梯度裁剪**：±5.0

### 训练过程
每个 batch：
1. 计算正负样本 FM 分数差 `diff = s_pos - s_neg`
2. BPR 损失：`-log(sigmoid(diff))`
3. 梯度系数：`grad_coeff = 1 - sigmoid(diff)`
4. 更新参数：w0, w_user, w_item, w_fine, v_user, v_item, v_fine

### 推理
- 对粗排 top 100 候选逐一计算 FM 分数
- 取 **top 30** 进入重排序

---

## 8. 重排序（多样性 + 新鲜度 + 来源加分）

### 策略（贪心选择）
每一步从剩余候选中选得分最高的，考虑：

| 因素 | 规则 |
|------|------|
| **品牌多样性惩罚** | 已出现过的品牌 -0.3 分 |
| **新鲜度加分** | 评价数 < 10 的商品 +0.05 分 |
| **多源加分** | 每个召回来源 +0.05 分 |

### 输出
- **最终推荐 top 10** 商品

---

## 9. 评估

### 评估指标
| 指标 | 说明 |
|------|------|
| **HitRate@10** | 推荐列表 top 10 中是否包含真实交互商品 |
| **NDCG@10** | 归一化折损累积增益（考虑排名位置和评分相关性） |
| **AUC** | 正样本分数高于负样本分数的比例 |

### 评估范围
- 随机选取 200 个有效测试用户
- 在**召回 → 粗排 → 精排 → 重排**各阶段分别计算指标
- 输出阶段间 HitRate 保留率（衰减分析）

### 健康检查项
- 数据：密度、平均交互数、评分分布、元数据覆盖率
- 划分：测试集占比、有效测试用户比例、冷启动比例
- 召回：Recall@1000、空召回率、各通道贡献
- 粗排：训练准确率、正负分离度、特征重要性分布
- 精排：BPR Loss 下降趋势、训练 AUC、正负分数分布、权重 NaN/Inf
- 重排：品牌多样性
